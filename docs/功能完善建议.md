## 二、P0 — 核心体验缺失（建议优先做）

### 1. 评论系统

博客最基础的互动功能，目前文章详情页没有评论区域。

**需要新增的模型：**

```python
class Comment(models.Model):
    """评论"""
    post = models.ForeignKey(Post, on_delete=models.CASCADE, related_name='comments')
    author = models.ForeignKey(User, on_delete=models.CASCADE, null=True, blank=True)
    nickname = models.CharField('昵称', max_length=50)  # 未登录用户可用
    email = models.EmailField('邮箱', blank=True)
    content = models.TextField('内容')
    parent = models.ForeignKey('self', null=True, blank=True, on_delete=models.CASCADE)  # 回复楼层
    is_approved = models.BooleanField('已审核', default=False)  # 后台审核制
    created_at = models.DateTimeField(auto_now_add=True)

    class Meta:
        ordering = ['-created_at']
```

**涉及改动：**

| 文件 | 改动 |
|---|---|
| `blog/models.py` | 新增 Comment 模型 |
| `blog/admin.py` | 注册 CommentAdmin |
| `blog/views.py` | 新增评论提交视图 + 评论列表查询 |
| `blog/urls.py` | 新增评论提交路由 |
| `templates/blog/detail.html` | 新增评论展示区 + 评论表单 |
| 侧边栏 `templates/blog/sidebar/comment.html` | 最新评论展示 |

### 2. 文章状态（草稿/发布）

目前创建即发布，无法保存草稿。读者可能看到未写完的文章。

**模型改动：**

```python
from django.db import models

class Post(models.Model):
    STATUS_CHOICES = (
        ('draft', '草稿'),
        ('published', '已发布'),
    )
    status = models.CharField('状态', max_length=10, choices=STATUS_CHOICES, default='draft')
    # ... 其余字段不变
```

**视图改动：**

- 前端只展示 `status='published'` 的文章
- 后台列表显示状态标签，支持筛选
- 后台增加「发布」和「存草稿」两个按钮

## 三、P1 — 重要功能，提升可用性

### 3. 文章缩略图 / 封面图

列表页和首页全是文字，缺少视觉元素。

**方案：** Post 模型加一个 `cover = models.ImageField(upload_to='covers/%Y/%m', blank=True, null=True)`，列表页展示缩略图，详情页展示封面大图。

### 4. RSS 订阅

博客标配功能，读者通过 Feedly / Inoreader 等工具订阅你的博客。

**Django 内置支持：**

```python
# blog/feeds.py
from django.contrib.syndication.views import Feed
from django.urls import reverse
from .models import Post

class LatestPostsFeed(Feed):
    title = 'MyBlog'
    link = '/'
    description = 'MyBlog 最新文章'

    def items(self):
        return Post.objects.filter(status='published')[:10]

    def item_title(self, item):
        return item.title

    def item_description(self, item):
        return item.desc

    def item_link(self, item):
        return reverse('blog:post_detail', args=[item.id])
```

只需加一个 URL 路由，不需要额外安装包。

### 5. 阅读时间估算

文章详情页显示「预计阅读 5 分钟」，提升用户体验。

```python
# Post 模型加一个方法
@property
def reading_time(self):
    """估算阅读时间（分钟）"""
    word_count = len(self.content)
    minutes = max(1, round(word_count / 300))  # 中文平均阅读速度 300字/分钟
    return minutes
```

### 6. 后台统计面板

目前只有文章总数和累计浏览，过于简单。

**建议增加：**

| 指标 | 实现方式 |
|---|---|
| 近7天发布趋势 | `Post.objects.filter(pub_date__gte=timezone.now()-timedelta(days=7)).count()` |
| 热门文章 Top 10 | `Post.objects.order_by('-pv')[:10]` |
| 分类分布 | 统计每个分类下的文章数，用图表展示 |
| 30天浏览量趋势 | 按天统计 pv 总和（需增加日 PV 记录模型） |

### 7. 文章导出

后台单篇文章或批量导出为 Markdown / HTML，方便备份或迁移。

## 四、P2 — 锦上添花

### 8. 友情链接

独立模型 + 后台管理 + 前端展示。

```python
class Link(models.Model):
    name = models.CharField('名称', max_length=50)
    url = models.URLField('链接')
    sort = models.IntegerField('排序', default=1, help_text='越小越靠前')
    is_visible = models.BooleanField('是否显示', default=True)
```

### 9. 站点地图 sitemap.xml

SEO 基础配置，让搜索引擎更快收录你的文章。

Django 自带 `django.contrib.sitemaps`，只需：

```python
# blog/sitemaps.py
from django.contrib.sitemaps import Sitemap
from .models import Post

class PostSitemap(Sitemap):
    changefreq = 'weekly'
    priority = 0.8

    def items(self):
        return Post.objects.filter(status='published')

    def lastmod(self, obj):
        return obj.pub_date
```

然后在根 URL 配置中加入 `sitemap.xml` 路由。

### 10. 标签云

侧边栏增加标签云展示，按文章数量决定标签字体大小。

### 11. 多级分类

目前分类是扁平的，不支持父子层级。可以用 `django-mptt` 或自引用外键实现。

```python
class Category(models.Model):
    name = models.CharField(max_length=32)
    parent = models.ForeignKey('self', null=True, blank=True, on_delete=models.CASCADE)
    # ...
```

### 12. 文章置顶

Post 模型加 `is_pinned = models.BooleanField('置顶', default=False)`，查询时 `order_by('-is_pinned', '-pub_date')`。

### 13. 图片集中管理

目前上传的图片散落在 `media/upload/img/` 目录下，没有统一管理界面。

**方案：** 做一个图片管理器页面，展示所有已上传图片，支持预览、复制链接、删除。

## 五、代码架构改进建议

### 5.1 拆分视图文件

当前 `blog/views.py` 273 行，前后端视图混在一起。

```
blog/
├── views.py              # 前端视图（index, detail, search...）
├── views_backend.py      # 后台视图（home_backend, add_kindeditor...）
├── views_api.py          # AJAX 接口（ajax_create_category...）
```
