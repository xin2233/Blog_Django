# 后台管理优化建议

## 一、当前后台现状

### 架构

```
backend_base.html  (Bootstrap 3 布局)
  ├── home.html             → 文章列表
  ├── add_kindeditor.html   → 新增文章（KindEditor）
  ├── add_ckeditor5.html    → 新增文章（CKEditor5）
  ├── add_wangeditor.html   → 新增文章（wangEditor）
  └── edit_kindeditor.html  → 编辑文章（KindEditor）
```

### 三套编辑器对比

| 页面 | 编辑器 | 状态 |
|------|--------|------|
| `add_kindeditor.html` | KindEditor | ✅ 完整（带分类/标签选择 + 图片上传） |
| `add_ckeditor5.html` | CKEditor5 | ⚠️ 只有编辑器，没有表单提交、没有分类标签 |
| `add_wangeditor.html` | wangEditor | ⚠️ 只有编辑器，没有表单提交、没有分类标签 |

### 优点

- 功能跑通了，能增删改查文章
- 尝试了多种富文本编辑器，有探索精神

### 不足

- **UI 较原始**：Bootstrap 3 有点过时，顶部导航栏就是一个蓝色条 + "后台管理" 文字，没有菜单层次
- **多套编辑器混用**：同一个项目里同时有三套编辑器，但只有 KindEditor 那套完整接入了表单，其他两套只是 "编辑器展示页"，没法真正提交文章
- **缺少通用后台功能**：没有分页、没有搜索筛选、没有批量操作、没有统计面板
- **Django 自带的 admin 没用到**：`admin.py` 里注册了 PostAdmin 但配置比较简单，CKEditor5 只配了 js/css，没有和 Django admin 的 `formfield_overrides` 结合

## 二、推荐的第三方后台管理框架

### 1. Django SimpleUI ⭐ 最推荐

```
pip install django-simpleui
```

- 基于 Vue + Element UI，界面现代、开箱即用
- 直接**增强 Django 原生 admin**，不需要重写前端页面
- 支持主题切换、菜单折叠、自定义图标
- 中文文档成熟，社区活跃
- 集成方式：只需加到 `INSTALLED_APPS`，现有的 `admin.py` 配置完全兼容

### 2. Django Grappelli

- 基于 Django admin 的皮肤增强
- 风格偏传统管理后台，比原生好看但不算现代
- 功能稳定，但多年未大更新

### 3. Django Jet / Jet Dashboard

- 类似 SimpleUI，但更新较慢
- 有 dashboard 面板，可展示统计图表

### 4. Xadmin（注意：Django 3.x 兼容性一般）

- 功能最丰富，但 Django 3.2 兼容有问题，需要自己修 bug

## 三、推荐方案

### 方案一（推荐）：SimpleUI + 只保留一套编辑器

```
┌─────────────────────────────────────┐
│  Django Admin + SimpleUI 皮肤       │  ← 管理后台界面
├─────────────────────────────────────┤
│  CKEditor5 / wangEditor 二选一      │  ← 富文本编辑器
│  (通过 formfield_overrides 集成)     │
└─────────────────────────────────────┘
```

- 使用 Django 原生 admin 的 `ModelAdmin` + `formfield_overrides` 配置富文本编辑器
- 手写的 `backend_base.html`、`home.html` 这些都可以废弃，SimpleUI 自带全套界面
- 只需要在 `admin.py` 里配置即可，不需要写视图和模板

### 方案二（轻量改动）：保留现有代码，但整理一下

如果不想引入大框架，至少建议：

- 三套编辑器只留**一套**（建议 wangEditor 或 CKEditor5，KindEditor 已停止维护）
- 把 `add_ckeditor5.html` 和 `add_wangeditor.html` 补全表单（现在只有编辑器，没有提交按钮和分类标签）
- `backend_base.html` 的 UI 可以换成 Bootstrap 5 或 Bulma（前端已经在用 Bulma 了，前后台统一风格）
