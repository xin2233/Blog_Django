# 数据备份与恢复操作说明

## 概述

博客系统支持两种方式进行数据备份与恢复：

- **方式一：后台 Web 页面**（推荐给不熟悉命令行的用户）
- **方式二：Django 管理命令**（适合服务器运维、脚本化、定时任务）

备份文件为 `.tar.gz` 格式，包含：
- `data.json` — 全量数据（文章、评论、用户、分类、标签等），通过 Django 的 `dumpdata` 导出
- `media/` — 用户上传的媒体文件（图片等）

> 支持所有 Django 支持的数据库后端：SQLite、MySQL、PostgreSQL 等。备份文件可在不同数据库之间迁移（例如从 SQLite 备份，恢复到 MySQL）。

---

## 方式一：后台 Web 页面

### 备份

1. 登录后台，进入 **系统 → 备份与恢复** 页面。
2. 点击左侧 **"立即备份并下载"** 按钮。
3. 浏览器自动下载一个 `.tar.gz` 文件（命名格式：`blog_backup_YYYYMMDD_HHMMSS.tar.gz`）。

### 恢复

1. 登录后台，进入 **系统 → 备份与恢复** 页面。
2. 在右侧 **"恢复数据"** 区域，点击 **"选择文件"** 选取之前备份的 `.tar.gz` 文件。
3. 点击 **"上传并恢复"**。
4. 在弹出的确认框中点击 **"确定"**。
5. 恢复成功后自动跳转到登录页面，请重新登录。

> ⚠️ 恢复操作会**覆盖**当前数据库和媒体文件，此操作不可撤销，请谨慎操作。

---

## 方式二：Django 管理命令

### 环境准备

```bash
# 进入项目目录，激活虚拟环境
cd Blog_Django
source .venv/bin/activate
```

### 备份

```bash
# 备份到默认目录（项目根目录下的 backups/）
python manage.py backup

# 备份到指定路径
python manage.py backup /path/to/backup_name.tar.gz
```

备份文件默认保存在 `backups/` 目录下，命名格式：`blog_backup_YYYYMMDD_HHMMSS.tar.gz`。

### 恢复

```bash
# 从备份文件恢复
python manage.py restore /path/to/backup_name.tar.gz
```

执行后终端会提示确认信息，输入 `yes` 后开始恢复。

> ⚠️ 恢复操作会**覆盖**当前数据库和媒体文件，此操作不可撤销。

## 迁移到新机器（完整流程）

假设需要将博客从 **机器 A** 迁移到 **机器 B**：

### 机器 A（导出）

```bash
cd Blog_Django
source .venv/bin/activate

# 方式一：生成备份文件
python manage.py backup

# 方式二：或者用 Web 页面下载

# 将备份文件拷贝到机器 B（例如通过 scp 或 U 盘）
```

### 机器 B（导入）

```bash
# 1. 部署项目代码（git clone 或复制代码）
git clone <仓库地址>
cd Blog_Django

# 2. 创建虚拟环境并安装依赖
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

# 3. 执行数据库迁移（创建表结构）
python manage.py migrate

# 4. 从备份文件恢复数据
python manage.py restore /path/to/blog_backup_20260720_120000.tar.gz

# 5. 启动服务
python manage.py runserver 0.0.0.0:8080
```

> ℹ️ 第 3 步必须先执行 `migrate` 创建表结构，再执行 `restore` 覆盖数据，否则恢复会失败。

## 自动化定时备份（可选）

可借助 Linux `crontab` 实现每日自动备份：

```bash
# 编辑 crontab
crontab -e

# 每天凌晨 3 点执行备份，保留最近 30 天的备份文件
0 3 * * * cd /path/to/Blog_Django && .venv/bin/python manage.py backup >> backups/backup_cron.log 2>&1
```

> 建议定期清理旧备份文件，避免占用过多磁盘空间。
