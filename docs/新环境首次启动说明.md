# 新环境首次启动说明

## 环境要求

- Python 3.x
- pip（Python 包管理器）

---

## 第一步：克隆项目

```bash
git clone <仓库地址>
cd Blog_Django
```

---

## 第二步：创建并激活虚拟环境

```bash
# 创建虚拟环境
python -m venv venv

# Windows (Git Bash / CMD)
venv\Scripts\activate

# Windows (PowerShell)
venv\Scripts\Activate.ps1

# Linux / macOS
source venv/bin/activate
```

激活后终端提示符前会显示 `(venv)`。

---

## 第三步：安装依赖

如果遇到 SSL 证书错误（`CERTIFICATE_VERIFY_FAILED`），使用国内镜像源：

```bash
pip install -r requirements.txt -i https://mirrors.aliyun.com/pypi/simple/ --trusted-host mirrors.aliyun.com
```

其他可选镜像源：

| 镜像源 | 地址 |
|--------|------|
| 阿里云 | `https://mirrors.aliyun.com/pypi/simple/` |
| 清华 | `https://pypi.tuna.tsinghua.edu.cn/simple/` |
| 中科大 | `https://pypi.mirrors.ustc.edu.cn/simple/` |

---

## 第四步：创建数据库迁移文件并建表

```bash
# 生成迁移文件（如果 blog/migrations/ 目录不存在，需先创建）
mkdir -p blog/migrations
touch blog/migrations/__init__.py

python manage.py makemigrations blog
python manage.py migrate
```

> **注意**：`blog/migrations/` 目录和 `__init__.py` 文件是必须的，否则 Django 不会为 blog 应用创建迁移。

---

## 第五步：创建超级管理员（可选）

```bash
python manage.py createsuperuser
```

按提示输入用户名、邮箱和密码。

---

## 第六步：启动开发服务器

```bash
python manage.py runserver
```

默认访问地址：**http://127.0.0.1:8000/**

其他常用方式：

```bash
# 指定端口
python manage.py runserver 8080

# 局域网可访问
python manage.py runserver 0.0.0.0:8000
```

---

## 常见问题

### 1. `ModuleNotFoundError: No module named 'django'`

虚拟环境未激活，或依赖未安装。先执行第二步和第三步。

### 2. `sqlite3.OperationalError: no such table: blog_post`

数据库表未创建。执行第四步的 `makemigrations` 和 `migrate`。

### 3. `UnorderedObjectListWarning`

传给 `Paginator` 的 QuerySet 没有排序。给 QuerySet 加上 `.order_by('-id')` 即可消除警告。

### 4. `pip install` 报 SSL 证书错误

使用国内镜像源加 `--trusted-host` 参数，参考第三步。
